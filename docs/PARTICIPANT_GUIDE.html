📋 Panduan Peserta

Setup Sebelum Workshop

Ikuti langkah-langkah ini di rumah sebelum hari-H. Jika stuck di mana pun, tanyakan saat workshop.

Cepat

Cara Cepat: Verifikasi Setup

Jika sudah clone repo dan install dependencies, cukup jalankan:

python check_setup.py
✅ Jika semua hijau

Anda siap! Tutup panduan ini dan santai sampai hari workshop.

❌ Jika ada yang merah

Ikuti langkah-langkah detail di bawah ini, atau datang 15 menit lebih awal di hari workshop untuk bantuan setup.

Persiapan

Langkah demi Langkah

Untuk peserta yang belum pernah setup atau mengalami masalah. Setiap langkah ada solusi jika gagal.

1

Install Python

Python adalah bahasa pemrograman yang kita gunakan. Versi minimal 3.11.

💡 Windows

1. Buka python.org/downloads
2. Klik "Download Python 3.11.x"
3. Jalankan installer
4. PENTING: Centang "Add Python to PATH" di bawah
5. Klik "Install Now"

💡 Mac

1. Buka Terminal (Cmd+Space, ketik "Terminal")
2. Ketik: python3 --version
3. Jika muncul versi (misalnya Python 3.11.4), selesai
4. Jika tidak, install dari python.org

Verifikasi:
python --version
⚠️ Jika "python is not recognized"

Windows: Coba py --version (bukan python)
Mac: Coba python3 --version (bukan python)

2

Install Git

Git adalah alat untuk mengunduh kode dari internet.

💡 Windows

1. Buka git-scm.com/download/win
2. Download dan jalankan installer
3. Terima semua default (klik Next, Next, Next...)

💡 Mac

1. Buka Terminal
2. Ketik: git --version
3. Jika belum ada, akan muncul prompt install. Klik "Install"

Verifikasi:
git --version
3

Install UV (Package Manager)

UV adalah alat untuk menginstall library Python. Lebih cepat dari pip.

# Windows (buka PowerShell):
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
# Mac (buka Terminal):
curl -LsSf https://astral.sh/uv/install.sh | sh
Verifikasi (tutup dan buka terminal baru):
uv --version
⚠️ Jika "uv is not recognized"

Tutup SEMUA jendela terminal, buka yang baru, coba lagi.

4

Clone Repository

Mengunduh kode workshop ke laptop Anda.

Buka terminal, lalu:
cd Documents
git clone https://github.com/asahfikir/pct-ai-workshop.git
cd pct-ai-workshop
⚠️ Jika "git clone" gagal

Periksa koneksi internet. Jika masih gagal, tanyakan URL yang benar ke instruktur.

5

Buat Virtual Environment

Virtual environment adalah "ruang terpisah" agar library workshop tidak bercampur dengan library lain.

Pastikan Anda di folder palembang-workshop, lalu:
uv venv

Anda akan melihat:

Using CPython 3.x.x
Creating virtual environment at: .venv
Activate with: .venv\Scripts\activate
💡 Aktifkan virtual environment

Windows: .venv\Scripts\activate
Mac: source .venv/bin/activate

✅ Cara tahu sudah aktif

Prompt terminal Anda akan menunjukkan (.venv) di depan, misalnya:
(.venv) C:\Users\Anda\Documents\palembang-workshop>

6

Install Dependencies

Menginstall library yang dibutuhkan (FastAPI, Uvicorn, dll).

Dengan (.venv) aktif:
uv pip install -r requirements.txt

Anda akan melihat daftar package yang diinstall:

+ fastapi==0.x.x
+ uvicorn==0.x.x
+ requests==2.x.x
+ pydantic==2.x.x
7

Buat Database

Membuat data akademik palsu (200 mahasiswa, 20 mata kuliah) untuk latihan.

Dengan (.venv) aktif:
python api/mock_database.py

Anda akan melihat:

Creating mock academic database...
Generated 200 students
Generated 20 courses
Generated 868 registrations
Generated 5243 grade records
...
Database created: academic.db
8

Verifikasi dengan check_setup.py

Script ini akan memeriksa semua komponen dan memberi tahu jika ada yang salah.

Jalankan:
python check_setup.py
✅ Jika semua hijau

Anda siap! Simpan terminal ini dan santai sampai workshop.

❌ Jika ada yang merah

Setiap error memiliki petunjuk perbaikan. Coba perbaiki, lalu jalankan ulang script. Jika masih stuck, datang 15 menit lebih awal di hari workshop.

Hari-H

Saat Workshop Berlangsung

Command yang perlu dijalankan saat sesi berlangsung.

✓

Terminal 1: Jalankan API Server

Buka terminal BARU (jangan tutup yang lama). Navigasi dan aktifkan venv:

cd Documents\palembang-workshop
.venv\Scripts\activate (Windows)
source .venv/bin/activate (Mac)
python api/main.py
⚠️ Jangan tutup terminal ini!

Biarkan berjalan di background. Jika ingin berhenti, tekan CTRL+C.

✓

Terminal 2: Jalankan MCP Server (Sesi 2)

Buka terminal BARU lagi:

cd Documents\palembang-workshop
.venv\Scripts\activate (Windows)
source .venv/bin/activate (Mac)
python mcp/server.py
?

Terminal 3: Testing

Untuk mencoba API:

curl http://localhost:8000/health
curl http://localhost:8000/stats/overview
💡 Windows: curl tidak ditemukan?

Gunakan browser: buka http://localhost:8000/docs dan klik tombol "Try it out".

Non-Teknis

Untuk Peserta Non-Teknis

Anda tidak perlu menjalankan kode. Fokus pada konsep.

💡 Apa yang Anda lakukan

1. Tonton demo instruktur di layar proyektor
2. Gunakan browser untuk test API: http://localhost:8000/docs
3. Tanyakan pertanyaan — perspektif Anda (manajemen, keuangan, dosen) sangat berharga
4. Fokus pada konsep: data terkunci → data terbuka → AI dapat bertanya

✅ Target Pemahaman

□ Apa beda chatbot dan agent?
□ Kenapa read-only lebih aman?
□ Apa itu MCP dan kenapa penting?
□ Pertanyaan apa yang harus saya tanyakan ke tim teknis?
□ Apa risiko kalau buru-buru ke produksi?

Bantuan

Masalah Umum & Solusi

!

"python is not recognized"

Windows: Coba py --version (bukan python)
Mac: Coba python3 --version (bukan python)

!

"uv is not recognized"

Tutup SEMUA terminal, buka baru, coba lagi.

!

"No module named 'database'"

Pastikan Anda di folder palembang-workshop (bukan di dalam api/).

!

Port 8000 sudah digunakan

Edit api/main.py, ubah port=8000 menjadi port=8002.

!

Semuanya gagal

Nuclear option: Hapus semua dan mulai dari awal:

cd Documents
rmdir /s /q palembang-workshop (Windows)
rm -rf palembang-workshop (Mac)
git clone https://github.com/asahfikir/pct-ai-workshop.git
cd pct-ai-workshop
uv venv && uv pip install -r requirements.txt
python api/mock_database.py
